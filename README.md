Описание проекта

Проект реализует две математические модели промышленного оборудования:

- Модель шлагбаума с управлением по Modbus TCP
- Модель узла смешивания воды с ПИД-регулятором и системой алертов

Вся инфраструктура контейнеризирована с использованием Docker и может быть развернута одной командой.

===============================================

Компоненты системы

1. Модель шлагбаума (Java + Modbus TCP)
2. Модель смешивания воды (Java + OPC UA)
3. OpenPLC - программируемый логический контроллер
4. Node-RED - шлюз для маршрутизации данных
5. InfluxDB - база данных временных рядов
6. Grafana - визуализация и система алертов

===============================================

Модель шлагбаума

Функциональность:

- Угол стрелы изменяется от 0 до 90 градусов
- Скорость движения 10 градусов в секунду
- Концевики срабатывают при 0 и 90 градусах
- Обработка ошибки при получении обеих команд одновременно
- Симуляция нестабильной сети (5% дублирование пакетов)

Протокол: Modbus TCP/IP

Адреса Modbus:
- Регистр угла: 0 (значение 0-90)
- Команда открыть: Coil 0
- Команда закрыть: Coil 1
- Концевик открыт: Coil 2
- Концевик закрыт: Coil 3
- Флаг ошибки: Coil 4

===============================================

Модель смешивания воды

Входные параметры:
- Расход горячей воды: 0-100 л/м (температура 80°C)
- Расход холодной воды: 0-100 л/м (температура 10°C)

Выходные параметры:
- Общий расход: сумма входов (л/м)
- Температура смеси: рассчитывается пропорционально

Логика работы:
1. Начальный нагрев до 80°C
2. При достижении 60°C отправляется алерт в Grafana
3. Активируется ПИД-регулятор
4. Подача холодной воды для охлаждения до 40°C
5. Стабилизация температуры на 40°C

Параметры ПИД-регулятора:
- Kp (пропорциональный): 2.0
- Ki (интегральный): 0.5
- Kd (дифференциальный): 0.1
- Заданная температура (setpoint): 40°C

Запуск проекта

Требования к системе:

- Docker и Docker Compose
- Java 11 или выше
- Git (для клонирования репозитория)

Шаг 1: Клонирование репозитория
```
git clone https://github.com/NikitaPut/industrial-digital-twin.git
cd industrial-digital-twin
```
Шаг 2: Запуск Docker контейнеров
```
docker-compose up -d
```
Проверка статуса контейнеров:
```
docker-compose ps
```
Все 4 контейнера должны быть в статусе "Up".

Шаг 3: Запуск модели шлагбаума
```
cd barrier-model
javac -d . src/main/java/com/industrial/BarrierFullModel.java
java com.industrial.BarrierFullModel
```
Шаг 4: Запуск модели смешивания воды

Откройте новый терминал и выполните:
```
cd industrial-docker/water-model
javac -d . src/main/java/com/industrial/WaterMixerModel.java
java com.industrial.WaterMixerModel
```

Управление моделями

Модель шлагбаума (команды в консоли):

o - открыть шлагбаум
Угол увеличивается на 10 градусов. При достижении 90 градусов срабатывает концевик открытия.

c - закрыть шлагбаум
Угол уменьшается на 10 градусов. При достижении 0 градусов срабатывает концевик закрытия.

e - тест ошибки
Симуляция получения обеих команд одновременно. Шлагбаум переходит в состояние ошибки и блокируется.

r - сброс ошибки
Возвращает шлагбаум в рабочее состояние.

s - показать статус
Отображает текущее состояние (угол, концевики, ошибка).

q - выход
Завершает работу программы.

Модель смешивания воды (автоматический режим):

Программа работает автоматически и не требует ввода команд. Последовательно выполняются этапы:

1. Нагрев от 20°C до 80°C
2. При 60°C: вывод сообщения об алерте
3. Активация ПИД-регулятора
4. Охлаждение до 40°C
5. Стабилизация температуры

Технические детали

Симуляция проблем сети

Модель шлагбаума включает симуляцию нестабильной сети:
- Вероятность дублирования пакетов: 5%
- При дублировании команда отправляется дважды с интервалом 50 мс
- Это позволяет тестировать устойчивость системы к сетевым ошибкам

Обработка взаимоисключающих команд

При получении обеих команд (открыть и закрыть) одновременно:
1. Устанавливается флаг ошибки
2. Шлагбаум блокируется
3. Движение стрелы прекращается
4. Для восстановления требуется команда сброса

Расчёт температуры смеси воды

Формула: T = (Горячая * 80 + Холодная * 10) / (Горячая + Холодная)

Пример:
- Горячая: 50 л/м, Холодная: 0 л/м -> Температура: 80°C
- Горячая: 0 л/м, Холодная: 50 л/м -> Температура: 10°C
- Горячая: 30 л/м, Холодная: 20 л/м -> Температура: 52°C

Возможные проблемы и решения

Проблема: Ошибка "Permission denied" при запуске Docker

Решение:
sudo usermod -aG docker $USER
Выйти и зайти заново

Проблема: Порты 3000, 1880, 8086 уже заняты

Решение:
Изменить порты в файле docker-compose.yml или остановить конфликтующие программы

Проблема: Не компилируется Java класс

Решение:
javac -version
Проверить установку Java. При отсутствии выполнить:
sudo dnf install java-17-openjdk-devel

Проблема: OpenPLC не отвечает на порту 502

Решение:
docker-compose restart openplc
docker logs industrial-openplc
