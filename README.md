## 📖 Описание проекта

Проект реализует **две математические модели** промышленного оборудования:

| Модель | Протокол | Назначение |
|--------|----------|------------|
| 🚧 Шлагбаум | Modbus TCP | Управление стрелой, концевики, обработка ошибок |
| 💧 Смешивание воды | OPC UA | ПИД-регулятор, температура, алерты |

**Особенности:**
- ✅ Полная контейнеризация через Docker
- ✅ Симуляция нестабильной сети (5% дублирование пакетов)
- ✅ Система алертов при достижении 60°C
- ✅ Готовые дашборды в Grafana

## 🚧 Модель шлагбаума

### Функциональность

- Угол стрелы: 0° — 90°
- Скорость движения: 10° в секунду
- Концевики срабатывают при 0° и 90°
- Обработка ошибки при одновременных командах
- Симуляция нестабильной сети (5% дублирование)

### Протокол: Modbus TCP/IP

| Адрес | Тип | Описание |
|-------|-----|----------|
| 0 | Register | Текущий угол (0-90) |
| 0 | Coil | Команда открыть |
| 1 | Coil | Команда закрыть |
| 2 | Coil | Концевик открыт |
| 3 | Coil | Концевик закрыт |
| 4 | Coil | Флаг ошибки |

## 💧 Модель смешивания воды

### Входные параметры

| Параметр | Диапазон | Температура |
|----------|----------|-------------|
| Горячая вода | 0-100 л/м | 80°C |
| Холодная вода | 0-100 л/м | 10°C |

### Выходные параметры

| Параметр | Формула |
|----------|---------|
| Общий расход | Q = Qг + Qх |
| Температура смеси | T = (Qг×80 + Qх×10) / (Qг+Qх) |

### Логика работы

```
Нагрев 20°C → 80°C
       ↓
 60°C → АЛЕРТ!
       ↓
Активация ПИД-регулятора
       ↓
Охлаждение → 40°C
       ↓
   Стабилизация
```

### Параметры ПИД-регулятора

| Коэффициент | Значение | Назначение |
|-------------|----------|------------|
| Kp | 2.0 | Пропорциональный |
| Ki | 0.5 | Интегральный |
| Kd | 0.1 | Дифференциальный |
| Setpoint | 40°C | Заданная температура |

## 🚀 Запуск проекта

### Требования

- Docker и Docker Compose
- Java 11 или выше
- Git

### Пошаговая инструкция

**1. Клонирование репозитория**

```bash
git clone https://github.com/NikitaPut/industrial-digital-twin.git
cd industrial-digital-twin
```

**2. Запуск Docker контейнеров**

```bash
docker-compose up -d
docker-compose ps
```

> Все 4 контейнера должны быть в статусе `Up`

**3. Запуск модели шлагбаума**

```bash
cd barrier-model
javac -d . src/main/java/com/industrial/BarrierFullModel.java
java com.industrial.BarrierFullModel
```

**4. Запуск модели смешивания воды**

Откройте новый терминал:

```bash
cd water-model
javac -d . src/main/java/com/industrial/WaterMixerModel.java
java com.industrial.WaterMixerModel
```

## 🎮 Управление моделями

### Модель шлагбаума

| Команда | Действие |
|---------|----------|
| `o` | Открыть шлагбаум (+10°) |
| `c` | Закрыть шлагбаум (-10°) |
| `e` | Тест ошибки (обе команды) |
| `r` | Сброс ошибки |
| `s` | Показать статус |
| `q` | Выход |

### Модель смешивания воды

**Автоматический режим** — программа работает без ввода команд:

1. 🌡️ Нагрев от 20°C до 80°C
2. 🚨 При 60°C → алерт в Grafana
3. 🔥 Активация ПИД-регулятора
4. ❄️ Охлаждение до 40°C
5. ✅ Стабилизация температуры

## 🔧 Технические детали

### Симуляция проблем сети

- Вероятность дублирования пакетов: **5%**
- При дублировании команда отправляется дважды (интервал 50 мс)

### Обработка ошибок

При получении обеих команд одновременно:
1. Устанавливается флаг ошибки
2. Шлагбаум блокируется
3. Требуется команда сброса (`r`)

### Расчёт температуры смеси

```
T = (Qг × 80 + Qх × 10) / (Qг + Qх)
```

| Qг (л/м) | Qх (л/м) | Температура |
|----------|----------|-------------|
| 50 | 0 | 80°C |
| 0 | 50 | 10°C |
| 30 | 20 | 52°C |
| 25 | 25 | 45°C |

## ⚠️ Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Permission denied (Docker) | `sudo usermod -aG docker $USER` + перезаход |
| Порты 3000/1880/8086 заняты | Изменить порты в docker-compose.yml |
| Java не компилируется | `sudo dnf install java-17-openjdk-devel` |
| OpenPLC не отвечает на 502 | `docker-compose restart openplc` |
